# Настройка системы аутентификации

## Что было реализовано

### Серверная часть (NestJS)

1. **Модель User** в Prisma схеме с полями:

   - `id` - уникальный идентификатор
   - `login` - логин пользователя (уникальный)
   - `password` - хешированный пароль
   - `created_at`, `updated_at` - временные метки

2. **Модуль аутентификации** (`src/auth/`):

   - `AuthController` - обработка запросов регистрации и логина
   - `AuthService` - бизнес-логика аутентификации
   - `JwtStrategy` - стратегия для JWT токенов
   - `JwtAuthGuard` - защита маршрутов
   - `auth.dto.ts` - валидация данных

3. **API endpoints**:
   - `POST /auth/register` - регистрация пользователя
   - `POST /auth/login` - вход в систему

### Клиентская часть

1. **Сервис аутентификации** (`src/utils/authService.js`):

   - Регистрация и логин
   - Управление JWT токенами
   - Валидация данных
   - Проверка аутентификации

2. **Обновленная страница логина** (`src/pages/LoginPage.js`):

   - Интеграция с новым API
   - Валидация на клиенте
   - Автоматический вход после регистрации

3. **Защита маршрутов** (`src/router.js`):
   - Проверка аутентификации для защищенных страниц
   - Автоматическое перенаправление на логин
   - Кнопка выхода в меню

## Настройка базы данных

### Вариант 1: Через SQL скрипт (рекомендуется)

Выполните SQL скрипт `server/create_users_table.sql` в вашей базе данных PostgreSQL.

### Вариант 2: Через Prisma (если PowerShell работает)

```bash
cd server
npx prisma db push
```

## Настройка переменных окружения

Убедитесь, что в файле `server/.env` есть:

```env
DATABASE_URL="postgresql://username:password@localhost:5432/your_database?schema=public"
JWT_SECRET="your-secret-key-here"
```

## Запуск приложения

1. **Сервер**:

```bash
cd server
npm run start:dev
```

2. **Клиент**:

```bash
cd client
npm run dev
```

## Использование

### Регистрация

1. Перейдите на `/login`
2. Введите логин (минимум 3 символа)
3. Введите пароль (минимум 6 символов)
4. Нажмите "Зарегистрироваться"
5. После успешной регистрации вы автоматически войдете в систему

### Вход в систему

1. Перейдите на `/login`
2. Введите логин и пароль
3. Нажмите "Войти"

### Защищенные страницы

- `/profile` - личный кабинет
- `/test-page` - страница тестов
- `/criteria` - оценочные критерии

### Выход из системы

Нажмите кнопку "Выход" в меню или перейдите на `/logout`

## Особенности реализации

1. **Безопасность**:

   - Пароли хешируются с помощью bcrypt
   - JWT токены с истечением через 24 часа
   - Валидация данных на сервере и клиенте

2. **Удобство использования**:

   - Автоматический вход после регистрации
   - Сохранение токена в localStorage
   - Автоматическое перенаправление на защищенные страницы

3. **Минимальная валидация**:
   - Логин: минимум 3 символа
   - Пароль: минимум 6 символов
   - Проверка уникальности логина

## Возможные проблемы

1. **Ошибки PowerShell**: Используйте SQL скрипт для создания таблицы
2. **Проблемы с CORS**: Убедитесь, что сервер запущен на правильном порту
3. **Ошибки базы данных**: Проверьте подключение к PostgreSQL
