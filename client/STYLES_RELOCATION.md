# Перемещение стилей из папки styles/

## Текущая структура

```
src/
└── styles/
    ├── base.css              # Базовые стили + утилиты
    ├── animations.css        # Анимации
    └── bootstrap-overrides.css  # Переопределения Bootstrap
```

## Варианты реорганизации

### Вариант 1: Переместить в `common/` (рекомендуется)

```
src/
└── common/
    ├── base.css
    ├── animations.css
    └── bootstrap-overrides.css
```

**Преимущества:**

- Логично: `common/` уже содержит общие компоненты (Page, Section, BasicModal)
- Все общие ресурсы в одном месте
- Простая структура

**Недостатки:**

- Смешивание JS и CSS файлов в одной папке

---

### Вариант 2: Разделить по назначению

```
src/
├── common/
│   ├── base.css
│   └── bootstrap-overrides.css
└── utils/
    └── animations.css  # Рядом с animations.js
```

**Преимущества:**

- `animations.css` рядом с `animations.js` - логично
- Базовые стили в `common/`

**Недостатки:**

- Стили разбросаны по двум папкам

---

### Вариант 3: Переместить в корень `src/`

```
src/
├── base.css
├── animations.css
└── bootstrap-overrides.css
```

**Преимущества:**

- Простота
- Все стили на одном уровне

**Недостатки:**

- Может быть беспорядок в корне `src/`
- Смешивание с JS файлами

---

### Вариант 4: Создать `assets/styles/`

```
src/
└── assets/
    └── styles/
        ├── base.css
        ├── animations.css
        └── bootstrap-overrides.css
```

**Преимущества:**

- Четкое разделение: стили в assets
- Масштабируемость

**Недостатки:**

- Дополнительная вложенность

---

## Рекомендация: Вариант 1

**Почему:**

- `common/` уже используется для общих ресурсов
- Простая и понятная структура
- Все общие стили в одном месте

## План действий (Вариант 1)

1. Переместить все CSS файлы из `styles/` в `common/`
2. Обновить импорты в `App.css`
3. Удалить папку `styles/`
